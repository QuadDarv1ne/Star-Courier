## 🔥 КРИТИЧЕСКОЕ ИСПРАВЛЕНИЕ (15.03.2026 22:30)

### Исправлена ошибка: `'dict' object has no attribute 'id'`

**Проблема:** При запуске новой игры возникала критическая ошибка `'dict' object has no attribute 'id'` в методе `run_dialogue()` (main.py:1027).

**Причина:** Функции `get_chapter_dialogues()` и `get_dialogue_by_id()` из `src/chapters_1_5.py` возвращали **словари** с данными диалогов, а метод `add_dialogue()` в `DialogueManager` и код в `main.py` ожидали объекты класса `Dialogue`.

**Решение:**
1. Добавлены функции конвертации в `src/chapters_1_5.py`:
   - `_convert_choice_to_object()` — конвертирует словарь выбора в объект `Choice`
   - `_convert_dialogues_dict_to_objects()` — конвертирует словарь диалогов в объекты `Dialogue`
2. Изменена `get_chapter_dialogues()` для возврата объектов `Dialogue` вместо словарей
3. Добавлена защита в `main.py:1024` — проверка `isinstance(selected_choice, dict)`

**Изменённые файлы:**
- `src/chapters_1_5.py` — добавлены функции конвертации (~90 строк)
- `main.py` — добавлена дополнительная проверка типа выбора

**Статус:** ✅ ИСПРАВЛЕНО — игра запускается, диалоги работают корректно
